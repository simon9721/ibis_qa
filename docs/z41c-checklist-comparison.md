## Executive Summary

The two workbooks are not equivalent outputs yet.

Micron's workbook is a compact, manually completed checklist. It summarizes model families, includes human judgement, and assigns an overall quality of `3MSX`. It explicitly records IBISCHK `7.0.2` with zero errors, warnings, and cautions.

Our generated workbook is an evidence-heavy automated report. It creates one sheet per component and one sheet per model, carries raw result evidence, includes Level 4 checks, and marks unresolved manual items as `MANUAL REVIEW`. The baseline comparison was made before the first follow-up fixes, when `z41c.ibs` reported `PASS=633`, `FAIL=34`, `WARN=100`, `NA=735`, `ERROR=0`, with candidate level `IQ1`.

After the implemented follow-up below, the regenerated `z41c.ibs` report is `PASS=646`, `FAIL=16`, `WARN=134`, `NA=708`, `ERROR=0`, with candidate level `IQ3`.

The largest differences are caused by:

- Micron applying engineering judgement, exceptions, and correlation designators that our tool does not yet ingest or apply.
- The generated workbook checking every model separately instead of family-level groups.
- The generated workbook including Level 4 Composite Current and ISSO checks, while the Micron workbook stops its model checklist at Level 3.
- Several current tool thresholds being intentionally strict, especially the 1 uA zero-current endpoint tolerance and simple semi-auto roughness/leakage triggers.
- Parser/modeling gaps in the tool, including scalar assignment parsing for `Vref = ...`, commented Model Spec evidence, bare-die pin RLC handling, and ISSO PU endpoint interpretation.

## Implemented Follow-Up

The first improvement pass addressed the highest-confidence tool-side gaps found by this comparison:

- `Vref = ...`, other `key = value` scalar forms, `R_load = ...`, and receiver-threshold assignment forms are now parsed.
- Bare-die components now report `3.2.2` as `NA` instead of failing per-pin RLC completeness.
- Input differential pins with `Tdelay=0ns` now pass `3.3.2`; missing/invalid `Vdiff` remains a hard failure and nonzero input `Tdelay` remains review evidence.
- `5.3.8`/`5.3.9` zero-current values outside the `1 uA` pass tolerance now report review-required `WARN` instead of hard `FAIL`, because the spec notes that special cases may not cross zero current at 0 V.
- Open-drain/open-sink missing pullup rows and open-source missing pulldown rows now emit explicit `NA` for applicable I-V checklist rows.
- Commented dynamic overshoot Model Spec rows are surfaced as semi-auto review evidence for `5.2.8`.
- ISSO PU endpoint comparison now uses `Ipu(Vcc)`, matching the equation stated in the check and the Micron DQ/DQS data.
- Target-level generation now supports `--max-level` / `--target-level`, explicit `NA` for no-result checklist rows, and optional GUI `*.review.json` overlays.

## Format Improvements Needed for Vendor-Style Comparison

1. Add a vendor-template mode.

   The generated spreadsheet should be able to produce a compact checklist with the same broad structure as the IBIS.org/Micron workbook: summary, component family, model family sheets, `PASS/FAIL/NA/EXCEPTION`, IQ level, exception, correlation, and comments.

2. Keep the current detailed mode.

   The per-component/per-model sheets and raw `results` sheet are very useful for debugging. They should remain available as an audit workbook or "detailed evidence" mode.

3. Add target IQ level filtering.

   If the target is IQ3, Level 4 checks should either be hidden from the checklist body or moved to an appendix. This would prevent Level 4 findings from obscuring comparison with an IQ3 vendor checklist.

4. Add review-decision overlays.

   The GUI already saves review decisions separately. The spreadsheet generator should consume those decisions and render accepted items as `PASS`, `NA`, or `EXCEPTION` with comments, while preserving raw generated evidence elsewhere.

5. Add designator support.

   The spreadsheet should explicitly support `S`, `M`, `G`, and `X` designators and their evidence. Micron's `3MSX` cannot be reproduced until the tool stores these reviewer decisions.

6. Add grouped model family summaries.

   The generated workbook should summarize by model family patterns such as `DQ_34_*`, `DQS_40_*`, `DM_*`, `TDQS_*`, and `ALERT_*`, while also allowing drill-down to individual parsed models.

## Recommended Next Steps

1. Implement a review overlay format and make spreadsheet generation consume it.
2. Use `--max-level 3` when reproducing this vendor checklist; the option also accepts the older `--target-level` alias.
3. Fix the likely tool bugs: scalar assignment parsing, bare-die `3.2.2`, `3.3.2` Tdelay handling, and ISSO endpoint equations.
4. Add a vendor-template spreadsheet mode once the review overlay exists.
5. Retest `z41c.ibs` and `z41c_it.ibs` together, because Micron's checklist covers both files.
